Challenge 5: Accessibility Concerns

Updates: I made the margin spacing smaller.

Challenge 2 is accessible to colorblind users because the white and black chess pieces are high contrast. Additionally, it is just a chessboard representation so there is no clicking necessary and thus not reliant on a mouse. However, there is also no alternative text since unicode characters are used in lieu of images. A possible solution may be to add a description of the chessboard at the bottom and provide hidden text behind each piece in addition to a labeled grid system so that assistive screenreaders can communicate positions.


Part 3 of Assignment 5: Find a new tag

I found the kdb tag, which is used to show keyboard commands. I searched for new types of inline elements and found one that seemed easy to use. I was surprised to see that it was an actual tag, not just some type in Courier font. I think this could be really useful on pages where people try to explain keyboard shortcuts. I might use it to leave myself a note about the shortcuts I can use in Atom like cmd+S to save as I work between the code and live server.
